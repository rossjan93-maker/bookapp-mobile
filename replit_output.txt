P2B.1 — statedReservation retrieval-provenance gate (architectural audit fix)
================================================================================

CONFIRMATION
------------
The architect audit identified that the original P2B `pickStatedReservation`
detector proved SCORING provenance (`stated_favorite:*` audit flag +
`stated_taste > 0`) but did NOT prove RETRIEVAL provenance (the candidate
was actually fetched by the P2A statedGenres branch in this pipeline run).
A book that arrived via revealedLanes / revealedAuthors / catalog branches
and merely happened to align with a stated favorite at scoring time would
have been wrongly reservable.

Fix shipped (3 files, 12-budget OK):

  1. lib/composition/statedReservation.ts
     Added retrieval-provenance gate: `book._retrieval_reason.startsWith
     ('stated_genre:')`. Runs FIRST in the candidate scan (cheap, fail-fast).
     Existing scoring-provenance gate runs SECOND. Both required — defense
     in depth, neither subsumes the other.

  2. scripts/validate_stated_reservation.ts
     Added 4 new cases (C9 / C10 / C11 / C12) + 2 sub-cases (C9b / C12b)
     covering the audit's exact drift case, exhaustive non-stated prefix
     sweep, strict AND-gate success, retrieval-only-without-scoring,
     catalog-source rejection, and mixed-pool selection.

  3. replit.md
     Appended P2B.1 bullet documenting the drift, the AND-gate fix, the
     soundness argument (planner branchOrder + first-seen-wins dedup at
     line ~1176 makes the `stated_genre:` prefix check reliable), and two
     known residuals NOT addressed by this patch (P2A-era author-branch
     reason hardcoding at line 705; absence of `_retrieval_reasons: string[]`
     multi-source provenance trail).

User decision honored: trace `reason` enum NOT split — both miss modes
collapse to existing 'no_eligible_candidate'.

TS-signature parity preserved: `pickStatedReservation` signature unchanged
(`_retrieval_reason` is already on `CandidateBook` / `ScoredBook`);
`RankedRecsResult.meta.stated_reservation_used` shape unchanged;
`STATED_RESERVATION_POLICY` constants unchanged.

Forbidden surfaces untouched: taxonomy, recPayloadCache/Session/Queue/
Validity, branch planner, recPrewarm, edit-preferences, schema, RecCard/
Home/TasteReadout, OL/GBooks/LLM, recommender pipeline geometry.
