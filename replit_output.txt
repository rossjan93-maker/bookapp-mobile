# P2DEBUG Instrumentation — Logs to Look For

All logs are `__DEV__`-gated, prefixed `[P2DEBUG/...]`. To remove later:
`rg -n "P2DEBUG" lib/ components/ app/` and delete each block.

## Instrumentation map (single source of truth)

| Tag                     | File                                       | Fires when                                              |
|-------------------------|--------------------------------------------|---------------------------------------------------------|
| `[P2DEBUG/save]`        | `app/edit-preferences.tsx`                 | User taps Save in Reading Taste editor                  |
| `[P2DEBUG/cause-set]`   | `lib/recRequest.ts` (setPendingBuildCause) | Save handler stamps the next-pipeline cause             |
| `[P2DEBUG/cause-consume]`| `lib/recRequest.ts` (consumePendingBuildCause) | Pipeline reads + clears the pending cause          |
| `[P2DEBUG/focus]`       | `components/RecommendationsFeed.tsx`       | For-You tab gains focus (every focus, gated or not)     |
| `[P2DEBUG/focus-trigger]`| `components/RecommendationsFeed.tsx`      | Focus passed all gates → pipeline will run              |
| `[P2DEBUG/build]`       | `components/RecommendationsFeed.tsx`       | RecRequest assembled — shows effective cause + signals  |
| `[P2DEBUG/plan]`        | `lib/retrieval/branchPlanner.ts`           | Branch planner finished — quotas + per-branch counts    |
| `[P2DEBUG/reservation]` | `lib/recommender.ts`                       | pickStatedReservation result (applied / reason / pick)  |
| `[P2DEBUG/topSlate]`    | `lib/recommender.ts`                       | Final composed slate — top4 ids/titles + retrieval src  |

## Test A — No refresh (save → back-nav → For-You)

Steps:
1. Open Reading Taste editor.
2. Add favorite_genres = [business], reading_styles = [fast_paced], avoid = [fantasy_scifi].
3. Tap Save. Wait for back-nav.
4. Land on For-You tab. Do **not** refresh the browser.

Expected log sequence (in order — note `cause-set` fires *before* `save` because
`setPendingBuildCause()` runs first inside the save handler):

```
[P2DEBUG/cause-set]     cause=explicit_preference_edit
[P2DEBUG/save]          cause=explicit_preference_edit  genres=[...]  styles=[...]  avoid=[...]
[P2DEBUG/focus]         hasTasteProfile=true tier=2 hasUserId=true hasSession=false
[P2DEBUG/focus-trigger] willRunPipeline=true
[P2DEBUG/cause-consume] cause=explicit_preference_edit
[P2DEBUG/build]         effectiveCause=explicit_preference_edit  consumedWas=explicit_preference_edit  configHash=<12-char>  statedGenres=[...]  softAvoids=[...]
[P2DEBUG/plan]          cause=explicit_preference_edit  stated=<n>  authors=<n>  lanes=<n>  fetchItems=<N>  byBranch=stated:X/authors:Y/lanes:Z  softAvoidLanesApplied=[...]
[P2DEBUG/reservation]   applied=true  reason=reserved  cause=explicit_preference_edit  key=<genre>  pickId=<id>  pickTitle="..."  compPoolSize=<n>  statedInPool=<n>
[P2DEBUG/topSlate]      cause=explicit_preference_edit  reservationApplied=true  reservationReason=reserved  top4=1.[<src>|st:0.??]"..." 2.[...] ...
```

The `<src>` token in `topSlate` is the prefix of `Book._retrieval_reason` — actual
values include `stated_genre`, `dominant_lane`, `liked_subject`, `local`, `repeated_author`,
`author_anchor`, etc. (NOT planner branch names like `revealedLanes`).

The `genres` / `styles` / `avoid` arrays in `[P2DEBUG/save]` reflect whatever the
chip components push into local state (canonical keys per the rest of Readstack;
treat this only as "did the values match what I tapped").

**Pass condition for Test A:** `effectiveCause=explicit_preference_edit` AND
`reservationApplied=true` AND at least one of the top4 entries has source prefix
`stated_genre`. Slot 1 is not guaranteed — the reservation seeds the comp pool
but final ordering can move other strong candidates ahead.

**Fail patterns:**
- `[P2DEBUG/focus]` shows `hasSession=true` → session wasn't cleared on save (regression in `clearRecSession` path).
- `[P2DEBUG/focus-trigger]` missing → focus gate (tier/userId/supabase) is blocking; check `tier`.
- `[P2DEBUG/build]` shows `consumedWas=null` → setPendingBuildCause never ran (save handler didn't reach line 179) or the consumer ran first.
- `[P2DEBUG/plan]` shows `stated=OFF` despite `statedGenres=["business"]` → planner gate bug.
- `[P2DEBUG/reservation]` shows `reason=no_eligible_candidate` despite `statedInPool>0` → reservation eligibility filter bug; if `statedInPool=0` despite `stated=<n>` in plan → retrieval didn't return business candidates.

## Test B — Refresh after save (save → back-nav → browser refresh → For-You)

Steps:
1–3 same as Test A.
4. Browser hard-refresh (Cmd-Shift-R / Ctrl-Shift-R).
5. Wait for For-You tab to render.

Expected log sequence (note `cause-set` precedes `save` for the same reason as Test A):

```
[P2DEBUG/cause-set]     cause=explicit_preference_edit
[P2DEBUG/save]          cause=explicit_preference_edit  ...
--- BROWSER REFRESH (module memory wiped) ---
[P2DEBUG/focus]         hasTasteProfile=true tier=2 hasUserId=true hasSession=false
[P2DEBUG/focus-trigger] willRunPipeline=true
[P2DEBUG/cause-consume] cause=null                          ← key signal: pending cause is gone
[P2DEBUG/build]         effectiveCause=session_open  consumedWas=null  configHash=<same 12-char as Test A>  statedGenres=["business"]  softAvoids=["fantasy_scifi"]
[P2DEBUG/plan]          cause=session_open  stated=<n>  ...
[P2DEBUG/reservation]   applied=false  reason=wrong_cause  cause=session_open  ...
[P2DEBUG/topSlate]      cause=session_open  reservationApplied=false  reservationReason=wrong_cause  top4=1.[revealedLanes|...]"<literary title>" ...
```

**Pass condition for Test B (confirms diagnosis D):** `consumedWas=null` AND `effectiveCause=session_open` AND `reservation.reason=wrong_cause` AND top4 slot 1 is NOT `stated_genre`.

This sequence proves the failure class is exactly:
> `setPendingBuildCause()` writes to module-only memory in `lib/recRequest.ts`. Browser refresh wipes that module, so the rebuilt pipeline sees `cause=session_open`, which fails the `STATED_RESERVATION_POLICY.allowedCauses` gate, so the top slate falls back to revealedLanes (literary).

---

## Cross-test invariants

- `configHash` should be **identical** between Test A and Test B (prefs unchanged between runs).
- `statedGenres` / `softAvoids` should be **identical** between Test A and Test B (proves signal compilation is correct in both cases — failure is purely the cause).
- `[P2DEBUG/cause-set]` will appear in Test B before refresh; the absence of a corresponding `[P2DEBUG/cause-consume]` with the same cause after refresh is the smoking gun.

---

## What this instrumentation does NOT cover

- AsyncStorage hydration (no fix attempted; that's the Phase 2.1 work).
- Scoring contributions / `_score_breakdown` deltas — `topSlate` shows `stated_taste` only as a coarse number.
- Explanation faithfulness — out of scope (P3).
- Retrieval-trace meta — already surfaced via existing `pipeline_meta`, not duplicated here.

## Removal

When the diagnosis is confirmed and the AsyncStorage fix lands:

```
rg -l "P2DEBUG" lib/ components/ app/
```

Then delete each `if (__DEV__) console.log('[P2DEBUG/...')` block (and the `_consumedCause` helper variable in `RecommendationsFeed.tsx` — collapse back to inline `cause: consumePendingBuildCause() ?? 'session_open'`).
