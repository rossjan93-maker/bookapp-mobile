Refactor the Profile experience to improve identity, onboarding correctness, and layout quality.

Do NOT redesign the whole app.
Do NOT add social systems.
Do NOT build the actual Reading Intelligence feature yet.
This is a profile and onboarding correction pass.

Goal
Make the Profile screen feel like a real personal reading identity page, not a sparse admin/settings page.

There are three required corrections:

1. username should be user-chosen, not auto-imposed
2. first and last name should be required at signup
3. Reading Intelligence should be reframed as a Coming soon section for now

--------------------------------
PART 1 — FIX SIGNUP IDENTITY MODEL
--------------------------------

Audit the current signup flow and profile data model first.

Then implement:

- First name is required at signup
- Last name is required at signup
- Username is a deliberate user choice, not auto-selected from signup input
- If a fallback internal handle is needed, keep it internal and do not present it as the user’s chosen public identity

If the user reaches profile without a chosen username:
- show a clear prompt to choose a username
- do not display a messy auto-generated public username as the primary identity

Output:
- files changed
- exact signup fields now required
- how username is now chosen and stored
- fallback behavior if username is missing

--------------------------------
PART 2 — REFRAME READING INTELLIGENCE
--------------------------------

Current issue:
- “Signals” and “Patterns” are being presented like a mature feature
- the content is too thin and makes the screen feel unfinished

Refactor this into a “Coming soon” presentation.

Requirements:
- Section title can remain “Reading Intelligence” if it reads well
- but the card(s) inside should clearly communicate:
  - this feature is coming soon
  - it will eventually surface signals and patterns from the user’s reading life

Preferred direction:
- one clean “Coming soon” card
- mention Signals and Patterns as future capabilities
- do not show weak placeholder intelligence cards pretending to be complete

Example direction:
Title: “Reading Intelligence”
Body: “Signals and patterns from your reading life — coming soon.”

--------------------------------
PART 3 — IMPROVE PROFILE LAYOUT
--------------------------------

Refactor the page hierarchy so it feels more organized and less bland.

Target structure:

Top:
- avatar
- full name
- chosen username
- key stats row

Middle:
- reading goal / reading summary block
- taste/profile setup CTA if relevant

Coming soon:
- Reading Intelligence card

Lower:
- friends
- recommendations sent
- settings

Requirements:
- stronger visual grouping
- clearer hierarchy
- less empty-feeling gray space
- more intentional spacing between sections
- profile should feel like a personal hub, not a settings dump

Do NOT overdesign.
Keep it calm, premium, and readable.

--------------------------------
PART 4 — COPY CLEANUP
--------------------------------

Make sure profile copy feels product-quality.

Avoid:
- weak placeholder language
- overly technical/system language
- awkward autogenerated identity display

Prefer:
- clean, human, editorial language
- honest “coming soon” framing
- clear personal identity presentation

--------------------------------
PART 5 — OUTPUT
--------------------------------

When complete, provide:
1. files changed

3. how username selection now works
4. what changed in profile layout
5. how Reading Intelligence is now presented
6. any small limitations intentionally deferred
