## 1. Extend BookTraits model

## 2. Unknown-first rule

This is critical.

If the evidence is weak, ambiguous, or absent:
- set trait to `unknown`
- do not infer aggressively
- do not impute from genre alone unless the rule is extremely conservative and documented

The purpose of P4B is not to make books seem richly understood. It is to create reliable evidence fields for later contributions.

## 3. Evidence sources

Use only existing metadata:
- title
- subtitle/series hints
- subjects
- description
- page_count
- existing enrichment fields
- existing deterministic helpers

No LLM calls.
No new external API calls.
No embedding system.
No semantic reranker.

## 4. Length class

Use deterministic page-count buckets.

Propose and implement conservative buckets, for example:
- short: <= 240 pages
- standard: 241–420
- long: 421–700
- tome: > 700
- unknown: no page count

If current project conventions suggest different buckets, justify them.

## 6. Tone / pace / complexity heuristics

Implement conservative deterministic heuristics.

Examples of possible signal classes:

Tone:
- dark: dark, bleak, grim, murder, grief, trauma, psychological, horror, noir
- light: funny, witty, humorous, heartwarming, cozy, uplifting, feel-good
- mixed: both strong light and dark evidence
- unknown: no strong evidence

Pace:
- fast: thriller, suspense, page-turner, action, fast-paced
- slow: slow-burn, contemplative, meditative, literary, reflective
- medium: some evidence but not strong
- unknown: no reliable evidence

Complexity:
- accessible: accessible, commercial, popular, cozy, beach read, self-help
- literary: literary fiction, lyrical, philosophical, reflective
- dense: dense, academic, complex, epic, experimental, history, theory, long-form nonfiction
- unknown: no reliable evidence

These examples are not prescriptive. Implement only what is safe.
