We need to change the onboarding architecture. The current order is wrong.

New principle:
Welcome the user into the app first.
Only after they understand the product should we ask for taste context and offer imports.

I want a two-phase onboarding model:

PHASE 1 — POST-SIGNUP APP WALKTHROUGH
Immediately after account creation, the user should be shown a short guided walkthrough of the app itself:
- what the core tabs/areas are
- what each one is for
- how navigation works
- where value lives
- what kinds of actions improve the experience over time

This should orient the user to the product before we ask them to do any intake.

PHASE 2 — RECOMMENDATIONS ENTRY PERSONALIZATION
Only when the user reaches the Recommendations tab should we ask them to personalize.
This is where we:
- encourage them to add context to their taste
- offer competitor-library imports as a high-visibility option
- explain that imports are the fastest path to strong recommendations
- provide a lighter preference intake for users who do not want to import yet

The current flow is doing too much too early and in the wrong order.

Target user journey:
1. Customer creates account
2. Customer is welcomed into the app with a short guided walkthrough of key functions and navigation
3. Customer lands in the app with a clear understanding of what exists
4. When the customer reaches Recommendations, they are met with:
   - an invitation to improve picks
   - a clear import option from Goodreads / StoryGraph / etc.
   - a lightweight taste intake option
   - the ability to skip and explore a starter deck if needed

This is the product shape I want:
Understand the app first.
Personalize when it becomes relevant.

This screen should make the value exchange obvious:
more signal -> better picks
