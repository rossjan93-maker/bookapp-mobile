Task type:
Bug investigation first, then minimal implementation only if root cause is clear.

Goal:
Fix Google sign-up/sign-in getting stuck on the “Signed in — loading your account…” screen in Replit desktop preview.

User-reported bug:
When signing up/signing in with Google in the Replit desktop preview:
1. Google auth appears to succeed.
2. A loading screen appears with the exact copy:
   “Signed in — loading your account…”
3. This loading screen never resolves.
4. If the user exits/closes that window, the Replit preview app shows a signed-in state/page, but still does not route forward into onboarding or the app.
5. User cannot continue without manually escaping the flow.

Expected behavior:
After Google auth succeeds, the app should automatically route to the correct next screen:
- onboarding/welcome/final setup for a new user
- app tabs for a returning/onboarding-complete user

No manual closing or exiting should be required.

Important context:
This appears to be an auth callback / bootstrap / routing handoff bug, not a recommendation or onboarding-content issue.

We have previously had onboarding/root-guard issues involving:
- local onboarding guard state
- durable DB onboarding completion state
- completeOnboarding()
- onboardingStage
- needsOnboarding
- root guard re-evaluation after auth state changes
