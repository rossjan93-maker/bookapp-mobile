P3A-6-A proved the composer-backed projection path behaves correctly under simulated flag-on and all validators passed. However, production default-on remains blocked because `_score_breakdown.explanation_quality` is still computed by the legacy phrasing-keyed classifier. Task type:
P3A-6-B internal classifier seam fix only.

Goal:
Add a contribution-based explanation-quality classifier and wire it behind the existing composer projection flag so that, when composer-backed reasons are enabled, `_score_breakdown.explanation_quality` is computed from `_scoring_contributions` rather than legacy reason phrasing.

This batch should not flip the composer default on.

## 1. Add contribution-based explanation quality classifier

Create:

- `lib/explanations/contributionQuality.ts`

or the equivalent file if the existing docs specify a path.

Export:

- `classifyContributionExplanationQuality(scoringContributions)`

It should classify quality using contribution evidence, not reason-string prefixes.

Suggested mapping from P3A-6-A:

- `stated_taste_fit` above floor + matchedKey present → `strong`
- `stated_taste_fit` above floor but generic → `acceptable_specific`
- `behavioral_fit` above floor with `genre_affinity` source → `acceptable_specific`
- `behavioral_fit` above floor but aggregate → `acceptable_generic`
- `feedback_fit` above floor → `acceptable_specific`
- `quality_reliability` only → `acceptable_generic`
- no above-floor supported contribution → `weak`

Please verify or refine this mapping based on the existing contribution kinds and display floors.
