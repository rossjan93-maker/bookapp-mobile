Desired onboarding shape:
The exact details are up to you, but directionally I want something like this:

Step 1 — Reader identity / orientation
- What kind of reader are you?
- fiction / nonfiction / both
- reading frequency
- current reading goal or cadence
- maybe whether they are trying to read more, read better, finish more books, discover better books, track reading, read with friends, etc.

Step 2 — Taste profile
Capture the things that actually shape fit:
- preferred genres / subjects
- disliked genres / subjects
- pacing tolerance
- tone preferences
- literary vs accessible
- plot-driven vs character-driven vs idea-driven
- series appetite
- length tolerance
- dark/heavy vs light/comforting
- experimental vs familiar
- etc.

Step 3 — Reading behavior / habits
- how they choose books now
- whether they abandon books often
- whether they reread
- whether they care about popularity vs niche
- whether they like being challenged vs staying in lane
- current formats if useful (audio/ebook/physical)

Step 4 — Optional historical grounding
- import history if available
- search and add past favorites if helpful
- BUT this should be optional and should not be the primary cold-start mechanism
- if you include book-based grounding, it should be lightweight and additive, not the main event

Step 5 — Guided product walkthrough
Show the user what the app does:
- recommendations
- tracking/progress
- timeline/feed
- friends/social
- explain briefly how feedback improves results
This should feel sequential and well-designed, not like dumping them onto a tab bar.

Step 6 — First recommendations handoff
- generate first recommendation stack
- explain why the stack looks the way it does
- make the “why this fits you” moment clear
- let the user start acting on recommendations immediately

- Strong defaults

- Clear progress indication
