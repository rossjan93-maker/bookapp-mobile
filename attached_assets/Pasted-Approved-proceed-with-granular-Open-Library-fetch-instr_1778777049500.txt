Do not implement behavior changes yet.

Instrumentation targets:
- fetchOLSubject
- fetchOLByAuthor

For each fetch attempt, log enough information to classify the outcome.

Required log prefix:
[OLFETCH/...]

Recommended log events:
1. Dispatch/start
   - kind=subject|author
   - query/subject/author
   - URL or a safe concise request descriptor
   - request start timestamp

2. HTTP response
   - status
   - ok true/false
   - elapsed ms

3. Parsed payload
   - raw docs.length before local filtering
   - if the payload shape is malformed, capture that as a parse/shape failure

4. Post-filter result
   - returned count after any current local filtering/normalization
   - if raw docs > 0 but returned count = 0, make that visible

5. Error path
   - error name
   - error message if safe
   - timeout vs fetch/network exception if distinguishable
   - elapsed ms

Critical requirement:
