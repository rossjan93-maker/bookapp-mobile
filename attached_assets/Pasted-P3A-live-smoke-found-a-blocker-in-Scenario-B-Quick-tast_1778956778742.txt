Problem statement:
The composer is not inventing a user preference — the user did state nonfiction. But it appears to be applying that stated nonfiction signal to books that are not nonfiction. That means the reason is user-grounded but not book-grounded.

P3A acceptance should require both:
1. the user signal exists, and
2. the book actually matches the cited signal.

Task type:
Targeted investigation only. Do not implement until root cause is pinned.

Goal:
Determine why fiction books received a composer-backed reason:
“Matches your stated nonfiction preference.”

Root-cause questions:

## A. Book metadata / catalog classification
Are The Fault in Our Stars or The Song of Achilles stored with:
- primaryGenre = nonfiction
- lane = nonfiction
- subjects/tags that map to nonfiction
- any broad subject that the mapper incorrectly treats as nonfiction?

If yes, classify as book-intelligence / catalog-metadata bug.

## B. Retrieval / branch provenance
Were these books retrieved via a stated_genre:nonfiction branch?
If yes, why did stated nonfiction retrieval produce fiction books?
Is the retrieval query too broad, or is Open Library / local corpus matching wrong?

## C. Scoring contribution derivation
Did `deriveScoringContributions(...)` attach `stated_taste_fit` with `matchedKey = nonfiction` merely because an audit flag existed, without validating book-side match evidence?

If yes, classify as contribution-evidence bug.

## D. Composer gating
Does `composeExplanation(...)` require both:
- user-stated key, and
- book-side match evidence?

Or does it trust the scoring contribution blindly?

If it trusts scoring blindly, determine whether the correct fix belongs:
- upstream in scoring contribution derivation,
- in book/lane classification,
- or as an additional composer-side book-truth gate.

## E. Nonfiction mapper
Is the nonfiction mapping too permissive?
Examples to inspect:
- myth / mythology / history / young adult / literary fiction / romance / biography-like subject tags
Could The Song of Achilles be mistakenly categorized as nonfiction because of ancient Greece / mythology / historical tags?
Could The Fault in Our Stars be miscategorized due subject metadata?
