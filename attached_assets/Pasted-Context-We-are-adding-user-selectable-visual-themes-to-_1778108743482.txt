PART 2 — Create a central theme registry

Create a maintainable theme registry based on tokens.json.

Suggested file structure:
- lib/themes.ts
or
- lib/theme/themeRegistry.ts
- lib/theme/ThemeProvider.tsx

Use semantic tokens rather than raw palette references wherever possible.

Each theme should expose semantic values such as:

- id
- name
- shortDescription
- mood
- colors:
  - appBackground
  - surface
  - surfaceElevated
  - cardBackground
  - textPrimary
  - textSecondary
  - textMuted
  - border
  - divider
  - accent
  - accentSoft
  - accentText
  - success
  - warning
  - danger
  - progressFill
  - progressTrack
  - tabActive
  - tabInactive
  - pillBackground
  - inputBackground
- typography:
  - displayFamily
  - bodyFamily
  - monoFamily if needed
  - headingWeight
  - bodyWeight
- radius:
  - card
  - pill
  - input
- shadow/elevation:
  - card
  - floating
- optional:
  - isDark
  - statusBarStyle

Requirements:
- tokens.json should drive the theme values.
- If tokens.json uses raw palette names, map them into semantic roles.
- If tokens.json already contains semantic roles, preserve them where possible.
- Adding a new theme later should be straightforward.

Acceptance criteria:
- no theme logic scattered across screens
- all themes are centralized
- app has a safe default theme

---

PART 3 — Theme provider and persistence

Add an app-level ThemeProvider.

Requirements:
- selected theme persists across app restarts
- use AsyncStorage for v1 unless there is already a strong user-preference storage pattern
- default theme should be North or the closest equivalent to the current app look
- theme loading must not block app boot
- failed theme load should safely fall back to default
- expose a hook such as useAppTheme() or useTheme()

Acceptance criteria:
- user selects a theme
- app changes theme
- user closes/reopens app
- selected theme remains active

---

PART 4 — Settings / Appearance theme selector

Add a Settings section for Appearance / Theme.

Requirements:
- Settings should include a clear Theme / Appearance row or section
- user can select North, Atelier, or Kiosk
- theme options should render as preview cards
- each preview card should show:
  - theme name
  - short descriptor
  - mini color palette
  - selected state
  - optionally a tiny sample card
- theme selection should apply immediately or after a simple confirmation
- selected state should be obvious

Acceptance criteria:
- theme picker is discoverable in Settings
- user can switch themes
- selected theme is clearly marked
- UI does not feel like a developer/debug tool
