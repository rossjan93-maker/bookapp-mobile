Mobile is the real requirement now.

We should stop optimizing around file download behavior on mobile.
The right mobile solution is: Share to readstack.

==================================================
PRODUCT DECISION
==================================================

For mobile, I want us to build a native share/import path so users can send Goodreads export data directly into readstack.

Target flow:
1. User opens Goodreads export page in mobile browser
2. If the export opens as raw CSV text or as a file, user taps Share
3. User selects readstack
4. readstack receives the shared payload
5. We detect whether it is Goodreads CSV
6. If valid, we import it directly or route to a confirmation/import screen

This should become the primary mobile solution.
