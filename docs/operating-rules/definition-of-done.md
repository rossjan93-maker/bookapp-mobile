# Definition of Done
A task is done only when:
1. It matches the acceptance criteria.
2. It works on device via Expo Go.
3. It does not break existing routes.
4. It does not add unapproved packages.
5. It does not change unrelated files.
6. It includes manual test steps.
7. If it changes schema, it includes a migration.
8. If it changes user behavior, it is behind a feature flag when appropriate.
